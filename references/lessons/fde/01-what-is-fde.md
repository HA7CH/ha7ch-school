# FDE · 第 1 课：FDE 到底是什么

> source: Lawted 本人口播稿/访谈（本地归档，见 `references/sources.md`）｜ 类型：讲授 ｜ 前置：无
> 这节课用 Lawted 自己的话讲，最有人味。教法遵循 `references/pedagogy.md`：像课本一样把概念讲透、给足，不盘问。

## 一句话本质（开场钩子）
**FDE 不是换了英文 title 的驻场外包，而是从一句模糊的"老板我也想用 AI，但不知道从哪开始"出发、亲手把它做成一线真的用得上的结果的那个人。**

## 为什么要在乎（钩子展开）
先勾住学生（尤其工程师）：
> "现在满世界都在说 FDE，售前改名叫 FDE、外包改名叫 FDE。这节课我不跟你玩名词，我给你一把尺子，让你自己能判断——一个岗、一个人、一件事，到底是不是 FDE。判准就四条，跟坐没坐在客户办公室一点关系都没有。"

## 核心概念（逐个讲透，内容给足）

### 1. FDE = Echo + Delta（贯穿全课的主框架，先立住）
Lawted 用两半拆一个人是不是 FDE：
- **Echo（摸真需求）**：进入陌生行业，找专家、跟老板和一线反复聊，把"我也想用 AI"这句表面需求，**追问成真问题**。
- **Delta（端到端做出来）**：调用 Agent 把这个问题**端到端**变成一个能运行、能上线、被人用的系统。
- **缺一半都不是 FDE**：只有 Echo = 咨询师/PM；只有 Delta = 等 PRD 的工程师。
> 这也是诊断口径：纯工程师往往 Delta 强 Echo 弱（习惯等明确 PRD、不愿见客户）；产品/运营/咨询的人往往 Echo 有底子、缺 AI-native 的 Delta。

### 2. 起点和终点，才是 FDE 与外包的真正分界（四条硬指标，本课的尺子）
别纠结坐没坐在客户工位。四条：
1. **起点是模糊问题**（老板一句"我想用 AI 但不知从哪开始"，没 PRD、连问题是什么都不清楚），不是确定方案；
2. **FDE 拥有问题定义权**，不只是负责执行；
3. **终点是业务结果**（跨境服装那单交付的不是 18 个网站，是新增客户，可以按有效客户结算），不只是功能完成；
4. **Coding Agent 把定制软件成本降一个数量级**，让一个人能干过去一个团队几百万的活。
> 四条都在 = 新物种；只改 title、仍按 PRD 写代码 = 换皮外包。**这就是为什么 2026 年 FDE 才成立：唯一的新变量是 Coding Agent 把定制那 30% 的成本压到接近 0。**

### 3. 反平台是生死线（土 FDE 的命门）
大厂平台型 FDE 背后一定有个要卖的平台（Palantir→Foundry、飞书 FDE→飞书、模型厂 FDE→API/token），天然会"把客户问题导向自己的产品"。**土 FDE 恰恰相反：技术中立、问题导向**——客户需要飞书就用飞书、十年前的技术能解决就用十年前的，先问"这个客户到底需要什么"，而不是"我手里有什么能卖"。
> Lawted 的加注：**对内可以标准化，对外永远交付 100 分的定制品**（那 30% 定制被 Agent 0 成本补齐）。谁把 FDE 做成"卖标准品/卖平台/更便宜的外包"，谁就死。

### 4. 价值在一线，不在拍板人（他最锋利的一刀）
> "你把 SaaS 丢给他，人根本都不会用；会用你 SaaS 的人自己 Claude Code 手术就脱出来了。所以你的价值在哪呢？还是在前期 FDE 进场理解业务、梳理工作流。"

价值不在那套软件、也不在拍板签单的老板嘴里，而在跟单员每天怎么用微信/Excel/PDF 复制粘贴的真实现场。**真正不可替代的是"判断问题、澄清需求、定义边界"**；写代码那部分，晚上转成 GitHub Issue 交给 Codex/Claude Code 去跑就行。技术实现越来越便宜，现场判断越来越贵。

### 5. 土 FDE vs 大厂 FDE（分层市场，不是谁取代谁）
- **大厂平台型**：有成熟平台、标准组件、合规/安全强，服务汇丰这类几百万几千万大单，最终目的是推动平台销售/续费。
- **土 FDE（创业型 / FDE OPC）**：直接面对大厂不经济服务的中小企业，几万到几十万预算，更轻更快，从一个很小的具体问题开始。
- 中国式 FDE 不是缩小版 Palantir，而是"大量懂行业、懂沟通、能借 Agent 独立交付的小型团队和超级个体"。
> 注意分层另一头：**项目一大**（大厂内部提效、国央企战略级）就不再是个人英雄，而是商务/产品/研发分工的**团队作战**——"小项目靠个人英雄，大项目最终一定是团队作战"。

### 6. FDE 是 High Agency 通才，不是全职权威专家
引李飞飞与 Mistral CEO 的判断：未来只剩两种人——1/1 的顶尖专家，和高主动性的通才（High Agency）。**FDE 属于后者：让专家能力真正落地的 High Agency 通才**，不是自己成为十年行业专家。关键专业判断（哪种医疗器械更安全）留给领域专家；FDE 优先做**跨行业相通的重复工作流**（录单/跟单/审核/字段提取）。对程序员尤其关键：代码变廉价，Skill 边界反而变大，必须走出办公室到前线交付。

## 讲完这节，你应该能答上（可选巩固 · 别逐题盘问、别当门槛）
1. **Q："FDE 不就是换了英文 title 的驻场工程师/外包吗"，你怎么反驳？**
   - 好答案信号：抓起点和终点、而不是纠结工位——起点是模糊问题（无 PRD）、FDE 有问题定义权、终点是业务结果（交付新增客户而非 18 个网站）、Coding Agent 把定制成本降一个数量级。答不出"问题定义权"和"为结果交付"就是只背了 title。
2. **Q：同样服务一家企业，大厂平台型 FDE 和土 FDE 会做出什么不一样的选择？为什么"反平台"是土 FDE 生死线？**
   - 好答案信号：大厂 FDE 背后有必须卖/续费的平台，会把问题导向自己产品；土 FDE 技术中立、问题导向，客户要什么用什么。能点出"对内标准化、对外 100 分定制""不能做成更便宜的外包"更佳。只会说"土 FDE 更便宜"是没懂。
3. **Q：一个纯后端工程师想转 FDE，你先让他补哪半？做过多年运营/PM 的人呢？**
   - 好答案信号：用 Echo/Delta 判断——工程师通常 Delta 强、Echo 弱（等 PRD、不愿见客户），逼他进现场练 Echo；运营/PM 往往 Echo 强、缺 AI-native Delta，补高强度用 Claude Code 端到端做真东西。能说出"价值在现场判断不在写每行代码"就真懂了。

## 常见误解（听到就纠正）
- ❌"把 FDE 当高薪 title 去追。" → 市场很快鱼龙混杂，真正区分你的是有没有进真实现场、有没有做出被人用的东西、有没有产生结果，不是名头/名校。Lawted 明说"不做 title 警察"。
- ❌"会写代码、会用 Claude Code 补几行就是 FDE。" → 只有 Delta 没有 Echo，顶多是等 PRD 的工程师。
- ❌"FDE 要打磨一个可复制的标准化产品/缩小版 Palantir。" → 对外永远交付定制结果；规模化靠"人的能力被复制"，不是把客户塞进同一套平台。
- ❌"FDE 的价值在向老板汇报、拍板签单。" → 价值在一线员工每天怎么干活的现场；靠谱 FDE 第一天梳理问题、第二天出流程、第三天出原型，不该两周都在写 PPT。

## 课堂练习（用四条硬指标做判断题）
给学生 3 段描述，让他用四条判断哪个是外包/土 FDE/大厂平台型，并说理由：
1. "按客户 PRD 给网站做多语言适配，验收收 5 万" → **外包**（起点是确定方案、终点是功能完成）。
2. "去货代老板现场，从他一句'我想用 AI'开始，梳理跟单流程做出单据数字员工，按处理效率/新增客户结算" → **土 FDE**（模糊起点+问题定义权+结果终点+Agent 独立交付）。
3. "飞书派驻工程师给已买飞书+AI 额度的工厂接安全帽/烟火识别" → **大厂平台型**（背靠平台、导向平台能力）。
做完让学生对**自己**做一次 Echo/Delta 自评：我更缺哪半、下一步进哪个现场补。
