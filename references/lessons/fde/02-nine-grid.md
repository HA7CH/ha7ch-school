# FDE · 第 2 课：一张图看懂 FDE 怎么做（九宫格）

> source: Lawted《一张图看懂 FDE 到底怎么做（FDE 九宫格）》口播稿+原图（本地口播稿归档）｜ 类型：讲授 ｜ 前置：`fde/01-what-is-fde`
> 这是 Lawted 自己发明、干了全国最大 FDE 社群（北上深杭四场百人线下会）总结出来的招牌框架。教法遵循 `references/pedagogy.md`：像课本一样讲透、给足。

## 一句话本质（开场钩子）
**FDE 九宫格 = 把"从模糊需求到可交付 AI 项目"拆成两个轴：纵轴是三种能力（商务 / 产品 / 研发），横轴是三层客户（30 人降本增效型 / 200 人进步型 / 国央企战略型）——3×3 共 9 格。客户层级越高，你的活儿就越从"做一个工具"走向"重构组织系统"。**

## 为什么要在乎（钩子展开）
上一课我们用 Echo+Delta 和四条硬指标讲了 FDE **是什么**。这一课讲**怎么做**——而且要让你明白一件反直觉的事：
> "同样是 FDE，面对 30 人的小厂、200 人的成长公司、和国央企，打法完全不同。用错层的打法，单子必死。这张九宫格就是让你先认清'这个客户在哪一层'，再决定商务怎么谈、产品怎么设计、研发怎么交付。"

## 核心概念（逐个讲透，内容给足）

### 1. 两个轴（先把坐标立起来）
- **纵轴 = 三种能力 / 角色：商务、产品、研发。** 这也是上一课说的 FDE 三轨度——出级 FDE 一个人全包，往上走就拆成团队分工。
- **横轴 = 三层客户，按"老板到底想要什么"分：**
  - **降本增效型（约 30 人）**——老板要**省钱**；
  - **进步型（约 200 人）**——老板要**进步**（不是省钱）；
  - **战略型（大型国央企）**——要**战略变革 + 安全合规可审计**。
- **一句话记住整张图的方向**：越往上（层级越高），越不是"做个工具"，而是"重构整个组织系统"。

### 2. 三层老板的心态（这是选层的关键，别搞错）
- **30 人老板**：极度务实。"你从他那儿转一块钱，他要从你这儿省出十块钱来。" 你要做的是快速回本、快速验证、建立信任。
- **200 人老板**：**不需要你降本增效**。他要的是"五年后回头看，今天做 AI 这件事是个转折点"。所以商务环节你要多讲**愿景**、AI 会如何改造他整体流程——讲省钱反而没打到点上。
- **国央企一把手**：要的是**整个公司战略层面的变化**。商务上你得抬头够大、能跟一把手/二把手对齐 mindset（Lawted 的真实案例：北京某国央企的 VP 常看他视频、还转给下面员工看，才谈得动）。

### 3. 第一层 · 降本增效型（约 30 人，一个人就能做掉）
老板说"我要用 AI"，本质是要省钱。三格：
- **⑦ ROI 切单（商务）**：明确省钱点、**聚焦高频痛点**、固定范围与报价、验收标准清晰。别接大而全，切一个能快速回本的小口。
- **⑧ 核心工作流梳理（产品）**：1-3 条关键流程（跟单 / 合同 / 对账这类人力密集的高频活）、输入输出明确。流程一般不多——它本来就是个高频痛点。
- **⑨ 轻量工具化（研发）**：插件 / 网页 / 轻 ERP、飞书机器人、简单自动化（含 RPA）。**必须快、稳、易用**——开发快、上手无学习成本，给天天看电脑的人用。
> 真实案例：Lawted 最早做的**物流那一单**就是这一层。

### 4. 第二层 · 进步型（约 200 人，一个团队分工做）
老板不为省钱，为组织进步。三格：
- **④ 共同定义试点（商务）**：识别核心意图、**锁定一个试点部门**（挑能被 AI 快速优化、快速出效果的）、成功标准定义、分阶段路线图（告诉他 D1/D2/D3/D4 各阶段 AI 帮他做成什么）。
- **⑤ 复杂工作流编排（产品）**：跨角色协作、识别 5-10 条工作流、把散乱数据流归一、数据流与依赖关系、异常处理设计、飞书 / ERP 集成。
- **⑥ 智能体工作系统（研发）**：给他上 Agent / WorkBuddy（Claude Code 这类），自研 Skill，**做一个统一知识库当"整个公司的业务大脑"**，所有人的 WorkBuddy 都连它；关键是**工作流的状态管理**。
> 真实案例：Lawted 做的**塞特车队**就是这型——不为裁员增效，而是求进步。方案用 Claude Code + ccconnect，分车队经理 / 公关 / 机械师 / 开发 Bot 等不同角色，统一知识库汇总品牌 logo / 公关文稿 / 车辆数据。

### 5. 第三层 · 战略型（大型国央企，要一个 FDE 团队）
这一层 Lawted 是到北京才接触到的（深圳/上海/杭州很少见）。要的是战略级立向 + 安全合规可审计。三格：
- **① 战略立项（商务）**：多意图识别、战略目标对齐、立项与预算路径。前提是你能对齐一把手 mindset。
- **② 企业级业务蓝图（产品）**：业务能力地图、**跨部门流程梳理**（财务/研发等多个部门，每个业务熟悉度都不同、极耗精力）、审计追溯、数据安全、合规与汇报语言、权限与责任设计、审批/审计节点、人机协同与评测。
- **③ 安全合规底座（研发）**：权限隔离、**省级安全隔离**、安全合规可审计——不能"出错了再改"，一切严格。这一层要上 **Ontology 级的严格建模**（Lawted 提到潘 Tim/T2 的 AutoLNG 那套为书记安全/合规设计的方法论）。
> 单子上百上千万，**绝不是一个 FDE 能做的，必须拉一个 FDE 团队**。

### 6. 横竖一起看：人才分级也在这张图里
- **横着三层，竖着三方面。** 第一层小单一个人做掉；第二层是一个团队、有分工（一人偏商务、一人偏产品、一人偏研发）；第三层战略级得整队作战。
- 对应人才培养：**初级 FDE** 做 30 人的（一个人）；**中级 FDE** 做 200 人的（团队，按商务/产品/研发三轨度要求）；**高级 FDE** 是去培训客户的 CTO / VP / 管理层。

## 金句（引用时标出处：Lawted《一张图看懂 FDE 到底怎么做》）
- "30 人的公司要快速省钱，200 人的公司想用 AI 推动组织进步，央国企则要求安全、合规与可审计。"
- "一个人能做小单，团队才能做工作台，战略级项目最终需要 Ontology 和完整的 FDE 组织。"
- "客户层级越高，工作就越从'做一个工具'走向'重构组织系统'。"
- "（30 人老板）你从他那儿转一块钱，他要从你这儿省出十块钱来。"
- "（200 人老板）他要的是五年后回头看，今天做 AI 这件事是个转折点。"

## 常见误解（听到就纠正）
- ❌"所有客户都用同一套打法。" → 三层心态完全不同：省钱 / 进步 / 战略。用 30 人的"省钱"话术去谈 200 人的进步型老板，直接脱靶。
- ❌"进步型也要主打降本。" → 200 人老板不为降本，商务要讲愿景和五年后的转折点。
- ❌"国央企也能一个人接。" → 战略级要 FDE 团队 + Ontology 级建模 + 安全合规底座，单子上百上千万。
- ❌"九宫格是让你把 9 格全做完。" → 它是**定位工具**：先判客户在哪一层，再决定这一层的商务/产品/研发各怎么打。

## 可以这样帮他自检（可选 · 不判分、别当考试）
用选择题给一个客户场景，让学生判断落在九宫格哪一层（选择题很适合这里）：
> 一家 30 人的货代，老板说"想用 AI"，其实天天在为跟单员肉眼对单、录单的人力发愁——这单该按哪一层打？
> 1. 降本增效型（⑦ROI切单 / ⑧核心工作流 / ⑨轻量工具化）
> 2. 进步型（试点 / 复杂工作流 / 智能体工作系统）
> 3. 战略型（战略立项 / 业务蓝图 / 安全合规底座）
>
> （答案 1：高频痛点 + 快速回本 + 轻量自动化。学生答完，点评他抓没抓住"老板要省钱、切高频痛点"这个层的特征。）
