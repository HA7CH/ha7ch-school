# FDE · 第 3 课：一线四城调查（2026 上半年北上深杭）

> source: https://www.ha7ch.com/writing/four-cities-fde-report ｜ 类型：讲授(文章) ｜ 前置：`fde/01`、`fde/02`
> 摸完自己、练完 Echo，这节抬头看整个市场。教法遵循 `references/pedagogy.md`。

## 一句话本质（开场钩子）
**四场线下、129 位 builder、单子从 ¥500 到 ¥2 亿——中国 FDE"有生意了，但还没有行业"。**

## 为什么要在乎（钩子展开）
> "前面你搞懂了 FDE 是什么、也被诊断了、也练了一把摸需求。现在最现实的问题：这门生意到底成不成？在中国能不能养活人？这节课不是理论，是 Lawted 跑了北上深杭四场闭门、面对面 129 个真在做的人，带回来的一手田野报告。"

## 核心概念（逐个讲透，内容给足）

### 1. "有生意，没行业"（本课的定盘星）
需求已经真实到能签 ¥500–¥2 亿的单、能招人发工资，**但**：报价拍脑袋、获客靠关系、没有标准交付物、没有人才培养体系。市场成熟到能赚钱，又不成熟到从业者连"该收多少钱"都不知道。这就是 2026 上半年中国 FDE 的真实坐标。

### 2. 四城画像差异（帮学生定位自己的城市/行业）
- **深圳**＝产业带/制造业+AI 混编，案例密度最高，重**物理结果**（裁员、人效、提成），还当场谈成校企项目制培养；
- **上海**＝金融重镇（资管/券商/合规）+国企，咨询式切入、¥2 亿信创大单，最强调**组织摩擦**（层层审批）；
- **杭州**＝最草根、电商为王，**价格谱最宽**（¥500 到十万+），转行者多（会计、操盘手转 FDE），供需错配最明显；
- **北京**＝资源密度最高，政企数字化、人才平台与课程创业、营收样本最大（几千万级），谈 PE 合资模式。

### 3. 护城河 = 行业业务知识，不是工程能力（本课最重要判断，重点敲）
纯工程能力正被 agent 抹平——谁都能写 agent；但懂某个行业"钱怎么流、活怎么干、卡在哪"的人稀缺。所以**会计、酒店老兵、操盘手转 FDE，可能比一个通用工程师更值钱**。给学生的信号：先补行业 know-how，别只堆技术。（这正好回扣第 1 课"价值在现场判断"。）

### 4. 定价与商业模式还在乱试
锚点各家不一：按数字员工年薪定价、按省下的人力成本倒推、按客户预算/焦虑程度定价（深圳原话"按客户焦虑程度报价"）、先收咨询费当筛子。模式有：项目制、订阅制、咨询先行、技术换股权、复用+token 计费、分阶段收费。很多人"不好意思收钱"，**工作流改善难以变现**是四城共同没答上的题。

### 5. 四层痛点（真正卡项目的不是技术）
- **需求层**：客户说不清痛点、给不出验收标准（只会说"要 AI 营销"）；
- **数据层**：企业数据不能直接喂 agent、敏感数据难打通、连客户都不信自己数据质量（**钢厂项目就死在数据质量恐惧上**），传统 RAG 精度在退化；
- **组织层**：员工抵制、老师傅怕被替、部门墙、效率一提高就引发裁员焦虑、内部改革者没议价权；
- **技术-商业层**：单任务成功率一串成完整业务链就崩、模型成本失控（有案例月消耗飙到 ¥3 万）、传统行业客户拖款、AI 咨询难规模化。

### 6. 六个 6–12 个月趋势判断
1) 定价快速收敛→低端（¥500–5K）先打价格战；2) **数据基建取代 agent 开发成 FDE 主战场**（知识库+数据能力能解 ~80% 项目问题）；3) 客户分层生死攸关（省钱叙事在国企失效，**增量市场如出海/外贸比在存量里砍成本更安全**）；4) 效率-裁员悖论从道德难题变成标准交付物（附带转岗方案）；5) 人才供给系统化（校企/课程认证，首批科班 FDE 2027 上半年入场，非技术路径可行）；6) 规模化路径决定资本是否进场（复用+token / 技术换股 / PE 合资三条路）。

## 讲完这节，你应该能答上（可选巩固 · 别逐题盘问、别当门槛）
1. **Q：作者说 FDE 现在最稀缺、最能定价的能力是什么？为什么？**
   - 好答案信号：答"行业/业务知识"，并解释纯工程正被 agent 抹平、谁都能写 agent，而懂某行业钱怎么流/活怎么干的人稀缺；能举转行者（会计/酒店老兵/操盘手）反而有优势。
2. **Q：报告识别出哪几层痛点？为什么真正卡住项目的往往不是技术？**
   - 好答案信号：说出四层（需求/数据/组织/技术-商业），并点明项目常死在数据质量和组织阻力（钢厂案例）而非模型做不出——数据基建与组织落地才是主战场。
3. **Q：四城生态各有什么不同？据此你会怎么选切入点？**
   - 好答案信号：能区分深/沪/杭/京四种画像，并结合"先按行业分层、增量市场比砍成本安全"给出自己合理的切入判断，而不是笼统说"到处都能做"。

## 常见误解（听到就纠正）
- ❌"FDE 就是更好的工程师。" → 纯工程被 agent 抹平，行业知识才是护城河。
- ❌"帮客户省钱东西就能卖出去。" → 省钱叙事在国企/保守行业失效；老板只认结果/成本/可复制，有时买的是安心。
- ❌"技术是最大瓶颈。" → 真正卡住的是数据基建和组织就绪度。
- ❌"国企单子比民企好做。" → 带详细 PRD 的国企单"易赚钱但没溢价"、几乎等同外包，执行还要层层审批。
- ❌"内部改革者能自下而上推动 AI 落地。" → 得一把手自上而下推；内部改革者没议价权、也没法把省下的效率变现。

## 课堂练习（收官作业，把整门课收回到"你能不能入局"）
让学生选一座最熟或最想切入的城市/行业（可以就是他所在城市），对照报告做一张一页纸：
1. 这个行业里有没有"能签 ¥500–¥2 亿"式的真实付费场景？写 2 个具体的；
2. 你在这个行业有什么别人没有的 domain know-how 或人脉？
3. 用报告里的定价锚点（数字员工年薪 / 省本倒推 / 客户焦虑 / 咨询筛子）给其中一个场景开一个报价，写清"凭什么收这个钱、能不能复制"；
4. 标出你预计会撞上的那一层痛点（需求/数据/组织/技术-商业）。
