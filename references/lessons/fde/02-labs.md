> 这是这门课的重头戏：把第 1 课的框架**照到学生自己身上**（Lab A），再让他**亲手练一次 Echo**（Lab B）。两个 lab 都要落回第 1 课的概念，别赶。

## Lab A · 被诊断一次（fde-pro）

### 目标
让学生被一台**诚实**的 FDE 适配诊断器完整看一遍：他现在到底是不是 FDE、缺的是 Echo（摸真需求）还是 Delta（端到端做出来）、该走土 FDE 还是大厂平台型、未来 30 天该干什么。产出是一份**挂着他简历原文证据**的个人诊断，不是通用鸡汤。

### 先打预防针（重要，先讲再做）
> "提前说清楚：这个 skill 背后是真正驻过场的 FDE 判断力，它**敢说'你现在不适合'**、敢说'你缺的不是 AI，是一个真实场景里被外部一线用过的产出'。它不是来夸你的。<50 分、被判'现在别碰'都是**正常且有价值**的结果——它会给你一张诚实的'起点卡'，不是'资格卡'。还有：别为了刷高分编假经历，它每个分都要挂你简历原文，喂空话反而扣。真正的爆点是被一个懂行的人看穿，不是被表扬。"

### 怎么拉起（用 Skill 工具）
调用 `Skill(skill="fde-pro")`，或对学生说触发词式的话（"帮我看简历能不能做 FDE / 我适合做 FDE 吗"）。fde-pro 已装在 `~/.claude/skills/fde-pro/`，一句话就能拉起。**它是引导式对话，不是一次性吐报告**，会先跟学生聊。

### 学生上手步骤
1. **准备输入**：简历（PDF/文本）或一段真实经历（做过什么项目、进没进过客户现场、有没有独立把一个东西端到端做出来）。没有简历也行——fde-pro 走口语化访谈，一次问一两题。
2. **第一步选路线**：它先用大白话讲两条路——大厂 FDE vs 土 FDE，按学生目标选；若经历跟所选明显不符，它会当场直说。
3. **交输入 / 答访谈**：关键探针别躲——**假需求题**（老板说"我要 AI 全自动"你怎么接）、**经济账题**（老板说一年省十几万你信不信）、**薄切片题**（半天时间+一份真单据你做什么让一线当场买账）。
4. **接住诊断**：它按十段释放——一句话判断 → 七维打分（每维挂原文）→ Echo×Delta 象限 → 优势/短板（敢点名）→ 简历怎么改成现金流叙事 → 未来 30 天动作 → 最适合你的第一个 FDE 项目。
5. **(可选) 出口三动作**：诊断完它会逐个问要不要 ①出一张 FDE 全息卡（`fde.ha7ch.com/c/{handle}`）②把简历做成 `cv.ha7ch.com` 在线版 ③把画像存进 HA7CH 岗位库意向。都不是必选，只想要诊断就停在诊断。

### 成功信号
学生拿到属于自己的诊断：一句可证伪的判断（如"你像工程师，不是能单干土 FDE 的人"）+ 七维分（每维挂着他自己简历/回答的原话）+ Echo×Delta 象限落点 + 一条贴着他象限的 30 天处方。他能在诊断里认出自己的具体经历被引用，而不是套话。

### 做完回扣第 1 课
问："诊断说你缺 Echo 还是 Delta？这和第 1 课那两半对上了吗？下一步你打算进哪个现场 / 补哪种能力？" —— **诊断结果直接写回学习存档 `interests`/`notes`，用来给后续排课**（缺 Delta → 引导去 AI Native 课）。

## Lab B · 货代实战擂台（练一次 Echo）

### 目标
把一次真实货代驻场 FDE 项目的"摸需求"半场，压成一局可玩挑战：学生进一间像素办公室，当那个被请来"捋一捋"的顾问，点 4 个各有心思的同事（老板/主管/业务员/跟单员）对话，从**相互矛盾、信息不对称**的说法里摸出这家 WAYBOUND 货代真正卡在哪——练的正是 FDE 的 **Echo**（推翻老板表面需求、价值去一线找）。

### 当前状态（如实告诉学生，别假装成品）
- ✅ 仓库**已能本地跑通**（`~/dev/fde-playground`）：不填 key 也能玩（NPC 走内置 mock 人设），想要真流式对话就在 `.env.local` 填 DeepSeek key。
- ⚠️ 边界要讲清：①这只是 **Phase 1「摸需求前门」**，只有 Echo 这半场，**还没有让你真去搭一个能跑切片的 Delta 半场**——完整的"2-3h 可评分驻场挑战"是项目目标，现在到的是前门。②"评分"是主管 LLM 对照真相的**定性点评 + 🟢🟡🔴 口头评级**，不是数字 rubric、没有排行榜。③按**本地** `http://localhost:3000` 玩（没有已确认的公开线上地址）。首次 `npm install` 或 DeepSeek 接口偶发慢/报错属预期，重试即可。

### 怎么起（导师带着跑，或让学生自己跑）
> 🔸 **先检测**：如果本机没有 `~/dev/fde-playground` 目录（你不是作者本人，只是装了这个 skill），这个 lab 暂时玩不了——**如实告诉学生"货代擂台还是作者本机的项目、没有公开可玩版本，我们先跳过它，只做 Lab A 诊断"**，别假装能起服务。检测到了再往下。

```bash
cd /Users/lawtedwu/dev/fde-playground
npm install
cp .env.example .env.local     # 想要真对话就在里面填 DEEPSEEK_API_KEY，不填走 mock
npm run dev                     # 起在 http://localhost:3000
```
浏览器打开 `http://localhost:3000`，点「▶ 进办公室」。

### 学生上手步骤
1. **进办公室，挨个点同事聊**：老板李总（里间）、主管阿强、业务员小敏、跟单员婷婷，再加几个干扰同事。每人只知道自己那摊、而且**挤牙膏**——问得泛就泛泛回，问到点子上才吐真话。
2. **盯右侧「线索笔记本」**：问对问题，关键信息自动记进去，★ 是核心痛点线索。目标是从矛盾里拼出流程全貌和真堵点。
3. **主动识破三个陷阱**：
   - ①老板嘴上要"AI 全自动"+张口说一年省十几万（**伪需求 + 算错的账**）；
   - ②跟单员的"细节漩涡"——问操作细节她能讲三段国别单证（ICS2/埃及 ACID/沙特 SABER），全是真的却一条线索不出，你得**从细节海里抬头抓框架**；
   - ③业务员说"发链接让客户填托书"——别家早有了，不算创新。
4. **把真痛点摸到手**：真答案是跟单员每天"对单"（开两个 tab 肉眼比 MBL/HBL/SI/补料）和"做账"最烧脑最赔钱、真价值在一线高频用户（不是主管自评那 1 小时）、该做**轻量 AI 增量插件而不是重做 ERP**。
5. **提交诊断**：集齐线索后点顶栏「📝 提交诊断」，写下你判断的真痛点 + 先做什么。主管阿强会对照真相点评并给口头评级。

### 成功信号
学生提交后拿到主管阿强对照真相的定性点评 + 🟢/🟡/🔴 评级，并且诊断里说出的是**真痛点**（对单/做账在赔钱、防错比多接单值钱、做轻量增量而非换系统），而不是被老板"AI 全自动"带跑；线索笔记本里 ★ 核心痛点大部分被点亮。

### 做完回扣第 1 课
问："你在这局里，有没有'推翻老板的表面需求'？真痛点是从老板嘴里听来的，还是从一线跟单员那儿摸出来的？这就是第 1 课说的'价值在一线不在拍板人'——你亲手体会到了吗？"
